# Centurion CRM

Real-estate deal pipeline and document workspace for **Greg / Centurion**. Built as a hosted Next.js web app: deals → contacts → templates with visual field bindings → generated documents.

> Design notes & roadmap live in `../we-re-going-to-make-merry-parnas.md` (a copy of `/Users/kam/.claude/plans/we-re-going-to-make-merry-parnas.md`).

## Quick start

```bash
npm install
npx prisma migrate dev               # creates ./dev.db
npx tsx prisma/seed.ts               # seeds Greg + assistant + one demo deal
npm run dev                          # http://localhost:3000
```

## Architecture

| Layer              | Choice                                          |
| ------------------ | ----------------------------------------------- |
| Framework          | Next.js 16 (App Router, Turbopack)              |
| UI                 | React 19 + Tailwind 4 + hand-rolled primitives  |
| ORM / DB           | Prisma 7 + SQLite (`better-sqlite3` adapter)    |
| Document engine    | `docxtemplater` (DOCX), `pdf-lib` (PDF)         |
| Drag & drop        | `@dnd-kit`                                      |
| Toasts             | `sonner`                                        |
| File storage       | `./public/uploads/` (swap to Vercel Blob in prod) |

See `lib/template-engine/` for the docx/pdf rendering pipeline and `lib/types.ts` for the `Binding` shape.

## v1 surface (this milestone)

- `/dashboard` — Kanban pipeline with drag-to-reorder
- `/deals`, `/deals/new`, `/deals/[id]` — list, create, detail (with inline edits + stage stepper + tasks + documents)
- `/contacts`, `/contacts/new`, `/contacts/[id]` — CRUD
- `/tasks` — overdue / today / this week / later
- `/templates`, `/templates/new`, `/templates/[id]` — upload DOCX/PDF and bind values to CRM fields
- `/deals/[id]/documents/new` — pick a template → produces a populated file
- `/inbox` — placeholder for Phase B Gmail intake

## Important Next.js 16 quirks

- `params` / `searchParams` are `Promise<…>` — always `await` them.
- `cookies()` and `headers()` are async.
- Page/Route prop types come from globals: `PageProps<'/deals/[id]'>`, `RouteContext<'/users/[id]'>`. They're generated by `next dev` / `next build` / `next typegen`.
- Tailwind 4 — no `tailwind.config.ts`; theme tokens go in the `@theme` block inside `app/globals.css`.

## Prisma 7 quirks

- `datasource db.url` is **not** in `schema.prisma` anymore — it lives in `prisma.config.ts`.
- Runtime client requires an explicit adapter; we use `@prisma/adapter-better-sqlite3`. See `lib/db.ts`.
- Generated client lives at `lib/generated/prisma/` (the `prisma-client` generator).
